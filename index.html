ChromePKG

The missing package manager for ChromeBooks

ChromePKG installs what you need with its dependencies:

$ crpkg install vim

It also registers the changes being made, so you can easily remove anything:

$ crpkg remove vim

See which packages are currently available:

$ crpkg search

Look for a package:

$ crpkg search vim

Update software lists:

$ crpkg update

Update ChromePKG packages:

$ crpkg upgrade

Install ChromePKG (along with Ruby and Git).

Available on aarch64, armv7l, i686 and x86_64.
curl -Ls https://git.io/JzMFm | bash

What is it?

ChromePKG is an open source package manager / source builder hybrid targeting Chromebooks and Chrome OS.

What does it do?

It installs the software you need that hasn't been provided by Google. Many important packages are already precompiled and it's enough to just type crpkg install package_name, but if something's not already there, you can easily build and install it from source.

How does it work?

In fact, ChromePKG is a simple Ruby script. There's also some Git involved, so we needed both of these things to run it on a bare Chrome OS. We have prebuilt them along with their dependencies to install into your system during installation. So, basically, after installing ChromePKG, you will have fully functional Ruby with Rubygems, Git and a package manager dedicated just for your Chromebook. Cool, huh?

How is it different from Crouton/Crostini?

Well, ChromePKG doesn't install an operating system. :p

The idea is that you may be on a weak internet connection and cannot download too much data, but you don't have Crouton and need just a few small packages. Also, you may be on a good internet connection and need just a few small packages. Also, why not use Chrome OS as the operating system?

Is GUI supported?

ChromePKG supports most GUI apps with help from the sommelier daemon. We don't support i686 architecture, due to Google dropping support.

How can I help?

If you have a compatible Chromebook, you can fork my Github repo and add new packages to the packages directory if you managed to build them from source successfully on your device. Package recipes are simple Ruby files - here is an example:
